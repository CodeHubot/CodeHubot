## 📖 文档说明

### 文档定位

**技术文档** 与 **开发规范** 的区别：

| 维度 | 开发规范 | 技术文档 |
|------|---------|---------|
| **定位** | 约束性规范 | 指导性文档 |
| **内容** | 代码风格、接口规范、命名规则 | 技术实现、架构设计、操作指南 |
| **受众** | 开发人员（强制遵守） | 开发+运维人员（参考学习） |
| **示例** | "API必须返回统一格式" | "如何实现JWT认证" |

### 适用对象

- **开发人员**：了解系统技术实现，快速上手开发
- **运维人员**：掌握部署运维流程，保障系统稳定运行
- **架构师**：理解系统架构设计，进行技术决策
- **新成员**：快速了解项目技术栈，融入团队开发

### 使用建议

1. **新成员入职**：建议按顺序阅读 02→01→05→03→04
2. **部署上线**：重点阅读 03_部署运维
3. **问题排查**：参考 04_开发调试/常见问题排查.md
4. **功能开发**：结合开发规范和对应的技术文档
